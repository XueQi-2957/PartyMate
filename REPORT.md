# PartyMate — 党务工作智能助手 · 调研报告

> 调研日期：2026年6月
> 调研人：基于高校学生党务负责人的实际工作经验 + 行业资料

---

## 一、调研背景

高校基层党务工作长期依赖人工操作，特别是在**异地校区**的管理模式下，信息传递不畅、文件审核耗时、流程标准化程度低等问题尤为突出。本报告聚焦高校学生党支部党务工作中的实际痛点，为后续 Agent 产品的功能设计提供依据。

---

## 二、高校党务工作的核心痛点

### 痛点1：发展党员材料审核 —— 最耗时、最容易出错

发展党员涉及 **5个阶段、20+份材料**，每一份都有严格的格式和内容要求：

```
申请入党 → 入党积极分子 → 发展对象 → 预备党员 → 正式党员
```

**常见具体问题：**

| 材料类型 | 常见错误 | 后果 |
|---------|---------|------|
| **入党申请书** | ① 落款日期与系统记录不一致<br>② 未用黑色钢笔/签字笔手写<br>③ 内容照搬网上模板，未结合个人实际 | 退档重写 |
| **思想汇报** | ① 标题格式不统一（有的写"思想汇报"，有的写"XX月思想汇报"）<br>② 没有结合当前政治热点（两会精神、主题教育等）<br>③ 字数不足（要求≥1000/1200/1500字不等）<br>④ 未手写签名 | 退回补正 |
| **入党志愿书** | ① "本人经历"时间段衔接不紧密（有断档）<br>② 家庭成员信息填写不完整<br>③ 介绍人意见与本人表现描述不一致 | 严重影响 |
| **考察表** | ① 考察意见过于笼统，没有结合具体表现<br>② 时间间隔不合规（如考察期不足半年）<br>③ 支部意见与个人考察内容逻辑矛盾 | 退回补正 |
| **转正申请** | ① 未在预备期满前提交（时间节点问题）<br>② 对预备期内缺点的改正情况阐述不够<br>③ 落款日期晚于支部大会讨论日期 | 延期转正 |

**现状：** 目前全部由人工逐份检查，一位党务负责人每周花 **3-5小时** 在材料审核上。遇到上级检查或毕业生档案移交时，时间更加紧张。

### 痛点2：三会一课学习内容组织 —— 缺乏系统性

"三会一课"（支部党员大会、支委会、党小组会、党课）要求定期开展：

| 类型 | 频率 | 难点 |
|------|------|------|
| 支部党员大会 | 每季度1次 | 学习主题选择、学习材料准备 |
| 支委会 | 每月1次 | 会议议题规划 |
| 党小组会 | 每月1-2次 | 讨论内容组织 |
| 党课 | 每季度1次 | 教学内容设计、PPT制作 |

**常见问题：**
- 学习内容来源单一（仅靠学校下发通知）
- 缺少与上级要求/时政热点的对应
- PPT制作耗时（每次2-4小时）
- 会议记录格式不规范，关键要素遗漏

### 痛点3：会议记录管理 —— 格式不一、存档混乱

每个会议需要记录：**时间/地点/参会人/主持人/记录人/议题/发言要点/决议/下次会议安排**

**常见问题：**
- 记录要素不完整（经常遗漏讨论过程、表决结果）
- 手写记录 + 电子记录双线并行，版本混乱
- 年底检查时发现缺项，无法补正
- 缺乏统一的存档命名规范

### 痛点4：党费收缴跟踪 —— 零散、易遗漏

| 问题 | 说明 |
|------|------|
| 计算复杂 | 不同身份（学生/教师）缴费标准不同 |
| 时间分散 | 每月收缴，但总有遗漏 |
| 记录追踪 | 收了多少、谁没交、谁交了现金，全靠手工记录 |
| 公示要求 | 需定期公示（格式有规范要求） |

### 痛点5：异地校区带来的特殊困难

| 问题 | 说明 |
|------|------|
| 通知滞后 | 学校文件传达到校区晚1-2天 |
| 材料递送 | 纸质材料需要跨校区递送 |
| 信息不对称 | 另一个校区的活动/要求经常不知道 |
| 办事成本高 | 简单的事也要往返两个校区 |

## 三、当前市场上的解决方案调研

| 产品/方案 | 类型 | 覆盖功能 | 不足 |
|-----------|------|---------|------|
| 学习强国 | 学习平台 | 学习内容、积分 | 不涉及材料审核和组织工作 |
| 智慧党建平台 | 管理系统 | 党员信息、组织关系 | 重管理轻审核，功能固化 |
| 各类党建APP | 移动端 | 通知、学习 | 功能单一，没有Agent能力 |
| 通用AI助手(ChatGPT等) | 对话 | 通用问答 | 不专业，不了解党务规范 |
| **结论** | | | **市场上没有专门针对党务工作痛点的AI Agent产品** |

## 四、Agent 方案可行性分析

### 为什么适合用 Agent 解决？

```
传统方案（规则引擎/脚本）：
  if 日期格式 == "XXXX年XX月XX日" → 通过
  else → 不通过
  → 只能检查格式，无法理解内容语义

Agent 方案（LLM+Tool）：
  读完整份材料 → 理解内容含义 → 
  结合时政知识判断思想汇报是否切题 →
  检查时间线是否合理 →
  返回结构化问题清单
  → 既懂格式又懂内容
```

### 收益量化

| 工作项 | 人工耗时 | Agent辅助后 | 节省 |
|--------|---------|------------|------|
| 发展党员材料审核（每次） | 2-3小时 | 5分钟检查+15分钟确认 | **90%** |
| 三会一课PPT制作 | 2-4小时 | 10分钟生成大纲 | **85%** |
| 会议记录整理 | 0.5-1小时 | 2分钟生成初稿 | **80%** |
| 学习内容收集整理 | 1-2小时/次 | 5分钟 | **90%** |
| **综合** | | | **~85%** |

## 五、MVP 功能定义

基于以上调研，MVP（最小可行产品）聚焦 **3个最高频、最痛点** 的功能：

### 功能1：发展党员材料合规检查（核心功能 🥇）

| 项目 | 内容 |
|------|------|
| **输入** | 入党申请书/思想汇报/转正申请等材料的文本或文件 |
| **检查维度** | ① 格式规范（日期、称呼、落款、标题）<br>② 内容完整（要点是否覆盖）<br>③ 时间线合理（关键节点间隔）<br>④ 交叉一致性（多份材料间是否有矛盾） |
| **输出** | 结构化问题清单：<br>❌ 错误项（需立即修改）<br>⚠️ 警告项（建议优化）<br>✅ 通过项（合规） |
| **技术难点** | 如何确保检查标准覆盖《发展党员工作细则》 |

### 功能2：三会一课内容准备助手（辅助功能 🥈）

| 项目 | 内容 |
|------|------|
| **输入** | 会议/党课主题（如"学习党的二十大精神"） |
| **输出** | ① 学习材料摘要<br>② PPT大纲（每页标题+要点）<br>③ 讨论题目建议 |
| **复用** | 直接调用现有的pptxgenjs生成PPT |

### 功能3：会议记录整理（辅助功能 🥉）

| 项目 | 内容 |
|------|------|
| **输入** | 语音转文字文本 / 手写记录拍照后的文本 |
| **输出** | 结构化会议纪要：<br>📅 基本信息（时间/地点/参会/记录人）<br>📝 议题列表<br>💡 决议事项<br>✅ 待办任务+责任人+DDL |

## 六、技术方案建议

| 层次 | 选择 | 理由 |
|------|------|------|
| **基础框架** | pydantic-ai | 轻量级Agent框架，单Agent+Tool模式，Pydantic类型安全 |
| **LLM调用** | 已有Hermes API | 无需额外配置 |
| **文件解析** | python-docx / pdftotext | 处理常见文档格式 |
| **输出格式** | Markdown + JSON | 既可视读又便于程序处理 |
| **开发语言** | Python | 与框架一致 |
