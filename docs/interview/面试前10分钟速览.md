## PartyMate 一句话

PartyMate 是一个本地优先的党务 Agent 工作台，核心是把大模型、RAG、工具调用、成员记忆、SQLite 台账和执行 trace 结合起来，让 Agent 不只是回答问题，而是能围绕材料审核、会议闭环和成员进度持续完成任务。

## MiniCode 一句话

MiniCode 是一个 Coding Agent 原型，用 Query Loop + Tool Use 实现多轮代码任务执行，并重点处理 Skill 路由、上下文压缩、记忆沉淀和高风险工具权限审查。

## 最容易被问的 12 个问题

### 1. 为什么做 PartyMate？

来自真实党务工作痛点。材料审核、会议纪要、成员阶段跟踪和待办提醒人工维护很容易漏，所以我把它做成一个本地优先、有状态、可追溯的 Agent 工作台。

### 2. 为什么它是 Agent？

它不是 prompt in / answer out。模型会拿到工具 schema，动态选择材料检查、知识库检索、成员查询、会议整理等工具，工具结果回填后继续推理，最后形成答案或回写任务。

### 3. Agent Runtime 怎么做？

手写轻量 ReAct loop：组装 system prompt、会话记忆、成员上下文和 tools schema -> 调 OpenAI-compatible chat completions -> 解析 tool_calls -> 执行白名单工具 -> observation 回填 -> 最多 10 轮 -> 输出最终回答并记录 trace。

### 4. 为什么不用 LangChain？

我了解 LangChain/LlamaIndex 生态，但 PartyMate 核心 runtime 是手写的。这样能更清楚地控制 tool calling、上下文组织、trace 和错误处理。复杂流程后续可以接 LangGraph。

### 5. RAG 怎么做？

文档清洗和分块后，用 ChromaDB 做向量检索，Ollama bge-m3 做 embedding，同时用 BM25 做关键词检索，再融合和 rerank，最后返回带引用的知识片段。

### 6. 为什么混合检索？

党务文档术语强、流程强。向量检索能处理语义相似，BM25 能抓住“发展对象”“政审”“预备党员”这类精确术语，混合检索更稳。

### 7. 成员级记忆是什么？

每个成员单独维护 memory namespace，保存阶段进展、风险提醒、审核结论和人工反馈。Agent 绑定某个成员时只加载这个成员的档案和记忆，避免上下文串扰。

### 8. Agent Trace 有什么用？

记录每次运行的用户输入、模型、工具调用链、参数、结果摘要、耗时和状态。它解决 Agent 黑盒问题，便于调试和审计。

### 9. 为什么用 Starlette？

PartyMate 是本地单用户 Agent 工作台，需要轻量 API、静态文件、异步处理、SSE 流式输出和 lifespan 初始化 MCP session。Starlette 比 Django 更轻，适合做 Web 胶水层。后续产品化可考虑 FastAPI。

### 10. 项目最大不足？

还不是生产级平台。RAG 缺系统 benchmark，workflow 还不是显式状态机，写操作确认和权限控制还能加强，前端也可以升级为 Vue/React。

### 11. MiniCode 的重点是什么？

重点是 Coding Agent 的执行循环和安全边界：Query Loop、Tool Use、Skill 路由、上下文压缩、记忆沉淀、路径和命令权限审查。

### 12. 你最大的优势？

我不是只会调模型 API，而是关注 Agent 的完整工程链路：工具 schema、上下文、RAG、记忆、错误恢复、权限控制、执行 trace 和业务闭环。

## 不要乱说的边界

1. PartyMate 核心 runtime 是手写的，不是 LangChain/LlamaIndex。
2. RAG 已有混合检索链路，但严格 benchmark 还没完整做。
3. PartyMate 是本地单用户工作台原型，不是成熟 SaaS。
4. Docker、LangGraph、FastAPI 可以说了解和后续计划，不要说已经深度落地。
5. 论文和科研贡献被问到时要如实说明自己的具体工作。
